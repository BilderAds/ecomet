# Session 2026-04-05-01

## Ziel

Ecomet Website komplett neu bauen (Modus A, Approach A) mit Next.js 16, Tailwind CSS 4, shadcn/ui, Framer Motion und Cobe Globe Animation. Alle Inhalte von ecometapp.de uebernommen.

## Kontext

- **Projekt**: [[projekte/ecomet]]
- **Modus**: A (Neubau) mit Approach A (React Komponenten, animiert)
- **Phase bei Start**: Nur Next.js Boilerplate, shadcn/ui initialisiert, keine Inhalte

### Entscheidungen

- Approach A (direkte React Komponenten) weil Dropshipper tech-affin sind
- Cobe Globe statt statischem Bild fuer Hero, zeigt China->DACH Versandrouten
- "Neue Bestellung" Popups auf Globe mit echten deutschen Staedten und Produkten
- TextScramble Animation verworfen, User wollte cleane Fade Animation
- Pain Points Section: Desktop expandierende Cards, Mobile eine Card mit Crossfade
- Copy "Mit ecomet laeuft es anders." als Uebergang zwischen Problemen und Loesungen (StoryBrand: Kunde ist Held)
- Headline in genau 2 Zeilen: "Schluss mit Paypal-Faellen und Retouren" + "4-8 Tage Versand & hoechste Produktqualitaet."

## Ergebnis

Website ist funktional komplett mit allen Sections. Build laeuft sauber durch. Desktop und Mobile optimiert. User Feedback: "sieht brutal aus", "schon geil geworden".

## Fehler & Probleme

- `Instagram` Icon existiert nicht in installierter lucide-react Version, musste Custom SVG verwenden
- Cobe v2 hat kein `onRender` Callback mehr, musste requestAnimationFrame + `globe.update()` nutzen
- Typografische Anfuehrungszeichen in JSX Strings verursachten Parsing Fehler
- Headline hat lange nicht in 2 Zeilen gepasst, mehrere Iterationen mit Font Size und Grid Layout noetig
- Globe war anfangs zu gross und hat Text in zu schmale Spalte gedrueckt
